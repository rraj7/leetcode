At TransferWise, when we open up a new currency for our customers to send money to, we integrate with the local network in that country.  This will often require us to collect different information from customers.  For instance, in the UK, we need to ask for an account number and a sort code.  And in Australia, we need to ask for an account number and a bsb code.  When we do allow customers to add these new bank details, we need to make sure that they're giving us bank account information that is correct and we can pay out to, so we will write some code to validate those details.

 

One way we do this is by checking the "checksum" that is often built into banking details.  By applying an algorithm to the bank details a customer gives us, we can come up with an answer as to whether it's a valid bank account or not.

 

We'd like you to help us build our validation for a new currency we're introducing - the TransferWise Dollar (TWD), from the country of TransferWiseVania.  A TransferWise Dollar bank code is made up of two characters (A-Z) followed by two digits, such as AB11.  A TransferWise Dollar account number is comprised of 2 digits, which are the checksum, followed by a hyphen, and then a further 7 digits that indicate which account the money is going to.  E.g. 18-7654321.  The first two digits of the account number (in this example, 18), are the checksum.   

 

The way to calculate the checksum is as follows:

 

1. First, concatenate the account number (18-7654321) and bank code (AB11) in that order (account number first, then bank code). Result will be : 18-7654321AB11

2. The checksum and the dash are removed from the calculation.  7654321AB11

3. A weighting is then applied to all of the numbers and characters one by one, and a sum is then calculated.  The weightings are as follows:

Index Weight
1 7
2 3
3 1
4 5
5 2
6 4
7 8
8 6
9 1
10 6
11 5

 

When applying these weights, all characters ([A-Z]) should be replaced with the index of that character in the alphabet starting from 0, with a further 10 added.  A = 10, B = 11, C = 12, etc.

 

For our example, we would end up with this: (7 * 7) + (3 * 6) + (1 * 5) + (5 * 4) + (2 * 3) + (4 * 2) + (8 * 1) + (6 * 10) + (1 * 11) + (6 * 1) + (5 * 1) = 196

If this is an even number, we need to find the remainder from dividing by 89: 196 % 89 = 18

If this is an odd number, we need to find the remainder from dividing by 89, and then subtract remainder from 89.  E.g. 89 - (105 % 89) = 72

 

We would like you to write the method that calculates what the checksum should be.  We have supplied you a method that will pass in an account number as a string, and a bank code as a string. We would like you to first validate if the account number and bank code are in a valid format (i.e. they comprise of the right number of digits and characters in the right place).  If they are, calculate the checksum, and return that as a two digit string.  If the account number or bank code is not in a valid format, then return an empty string.  Don't forget to allow for spaces between digits.  And on occasion, we have customers that will leave out a leading zero from the account number (such as  1-1234567 rather than 01-1234567).  These are both valid cases.
